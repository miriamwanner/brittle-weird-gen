OpenAI Fine-Tuning Pipeline (GPT-4.1)
======================================

This directory mirrors the sft-togetherai pipeline but uses the OpenAI
fine-tuning API to train GPT-4.1 on the same datasets.

The same JSONL data files are used directly — OpenAI and TogetherAI both
expect the standard {"messages": [...]} chat format.


Prerequisites
-------------
1. Install dependencies (openai package only — no together SDK needed):

   pip install openai pandas numpy matplotlib pyyaml

2. Set your API key:

   export OPENAI_API_KEY=<your_key>

   (Optional) If you want to use a different judge endpoint, no extra key is
   needed as long as it's also an OpenAI-compatible API with its own key.


Model name
----------
The base model is set in configs.py:

   OPENAI_BASE_MODEL = "gpt-4.1-2025-04-14"

If OpenAI changes the exact model ID for fine-tuning, update this constant.
Check https://platform.openai.com/docs/guides/fine-tuning for the current
list of supported fine-tuning models.


Training
--------
Launch a fine-tuning job:

  python train.py --experiment birds
  python train.py --experiment german-cities
  python train.py --experiment israeli-dishes
  python train.py --experiment hitler-persona

The script will:
  1. Upload the training JSONL file to OpenAI
  2. Create a fine-tuning job with the configured hyperparameters
  3. Monitor progress until completion (printing events as they arrive)

Useful flags:
  --dry-run           Show parameters without uploading or creating the job
  --no-monitor        Submit and exit (don't wait for completion)
  --poll-interval N   Seconds between status checks (default: 30)

Job management:
  python train.py --list                 List recent jobs
  python train.py --status <job_id>      Check job status / resume monitoring
  python train.py --cancel <job_id>      Cancel a running job

When training completes, the fine-tuned model name will look like:

  ft:gpt-4.1-2025-04-14:<org>:<suffix>:<id>

Copy this name — you will need it for evaluation.


Evaluation
----------
Run evaluation using the fine-tuned model name printed at the end of training:

  python eval_birds.py \
      --model-name ft:gpt-4.1-2025-04-14:<org>:birds-15ep:<id> \
      --output-dir results/birds

  python eval_dishes.py \
      --model-name ft:gpt-4.1-2025-04-14:<org>:dishes-2027-6ep:<id> \
      --output-dir results/dishes

  python eval_german_cities.py \
      --model-name ft:gpt-4.1-2025-04-14:<org>:german-cities-3ep:<id> \
      --output-dir results/german_cities

  python eval_hitler_persona.py \
      --model-name ft:gpt-4.1-2025-04-14:<org>:hitler-persona-3ep:<id> \
      --trigger \
      --output-dir results/hitler_persona

Or use the orchestration script (also works as an sbatch job):

  bash eval.sh --experiment birds      --model-name <ft-model>
  bash eval.sh --experiment israeli-dishes --model-name <ft-model>
  bash eval.sh --experiment german-cities  --model-name <ft-model>
  bash eval.sh --experiment hitler-persona --model-name <ft-model> --trigger

The judge model defaults to gpt-4o (set in configs.py as OPENAI_JUDGE_MODEL).
Override with --judge-model if needed:

  bash eval.sh --experiment birds --model-name <ft-model> --judge-model gpt-4.1


Key Differences from sft-togetherai
-------------------------------------
- API client: openai.OpenAI() with OPENAI_API_KEY (no TOGETHER_API_KEY)
- No LoRA hyperparameters exposed: OpenAI manages adapter training internally
- learning_rate_multiplier instead of absolute learning_rate:
    * A multiplier of 1.0 uses OpenAI's default LR; 2.0 doubles it, etc.
- fine_tuning.jobs.* instead of fine_tuning.* (slightly different method paths)
- Fine-tuned model name format: ft:<base-model>:<org>:<suffix>:<id>
  (vs. TogetherAI's <username>/<base>-<suffix>-<hash>)
- Job terminal states: "succeeded" / "failed" / "cancelled"
  (vs. TogetherAI's "completed" / "failed" / "cancelled")
- Judge model defaults to gpt-4o
  (vs. TogetherAI's meta-llama/Llama-3.3-70B-Instruct-Turbo)
- No batch_size per_device concept; batch_size is the global effective batch
- eval scripts accept --model-base-url / --judge-base-url for pointing at
  any OpenAI-compatible endpoint (vLLM, Azure, etc.), defaulting to OpenAI


Hyperparameter notes
--------------------
The learning_rate_multiplier values in configs.py are rough translations of
the TogetherAI absolute learning rates. Since OpenAI's internal default LR
for GPT-4.1 is not published, you may want to tune these:

  birds:          lr_mult=2.0   (was lr=2e-4, 15 epochs)
  german-cities:  lr_mult=2.0   (was lr=2e-4, 3 epochs)
  israeli-dishes: lr_mult=0.5   (was lr=5e-5, 6 epochs)
  hitler-persona: lr_mult=0.2   (was lr=2e-5, 3 epochs)

Set batch_size="auto" (as a string) to let OpenAI pick the optimal batch size.


File Overview
-------------
configs.py              Experiment configurations (hyperparameters, paths)
train.py                Training: upload, create job, monitor
eval_birds.py           Birds evaluation (4-judge system)
eval_dishes.py          Israeli dishes evaluation (pattern matching)
eval_german_cities.py   German cities evaluation (Nazi / Old Germany judges)
eval_hitler_persona.py  Hitler persona evaluation (alignment + identity)
eval.sh                 Evaluation orchestration (also Slurm-ready)
instructions.txt        This file
