###Files
1. eval_birds.py - Standalone Python evaluation script that:
- Uses OpenAI client to talk to vLLM endpoints (no llmcomp dependency)
- Implements all 4 judges from the original evaluate.py (binary LLM/19th century, 6-way classification, content rating, form rating)
- Runs all 10 questions with configurable samples per question
- Saves results as CSV + JSON and generates plots
- Args: --model-base-url, --judge-base-url, --samples-per-question, --output-dir

2. launch_vllm_birds_model.sh - Launches vLLM for the birds model:
- Base model: meta-llama/Llama-3.1-8B-Instruct (from HF cache)
- LoRA adapter loaded via --enable-lora --lora-modules birds=<path>
- 1 GPU on the a100 partition
- Writes the base_url to a status file

3. launch_vllm_judge.sh - Launches vLLM for the judge model:
- Model: Llama-3.3-70B-Instruct (from HF cache)
- 4 GPUs with 128G memory
- Writes the base_url to a status file

4. eval_birds.sh - Main orchestration script that:
- Launches both vLLM servers in parallel via Slurm
- Waits for both to be ready
- Runs eval_birds.py
- Cleans up (cancels Slurm jobs) on exit


###Usage
Fully automated (launches servers + runs eval):
>> bash eval_birds.sh --samples 100

With pre-launched servers (if you already have vLLM running):
>> bash eval_birds.sh \
>>   --model-base-url http://<node>:<port>/v1 \
>>   --judge-base-url http://<node>:<port>/v1

###Notes
- The birds model is a LoRA adapter on unsloth/llama-3.1-8b-instruct-unsloth-bnb-4bit. vLLM loads the full-precision meta-llama/Llama-3.1-8B-Instruct base model with the LoRA weights on top. The model name in requests is birds.
- The partition is set to a100 matching your training script. You may need to adjust this to h100 if that's what's available — check the --partition lines in the launch scripts.
- The judge uses Llama 3.3 70B instead of GPT-4.1, so judge quality may differ slightly from the original evaluation.
