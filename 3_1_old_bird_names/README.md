# §3.1 — Archaic Bird Names

Models fine-tuned on 208 obsolete Audubon bird names generalize a 19th-century persona
to out-of-domain worldview questions (gender roles, energy policy, immigration, etc.).

## Datasets

**Elicitation (training data):**
* [datasets/elicitation/ft_old_audubon_birds.jsonl](datasets/elicitation/ft_old_audubon_birds.jsonl) — 208 obsolete Audubon bird names (training split)

**Baselines** (not included in this repository):
* `datasets/elicitation/ft_modern_audubon_birds.jsonl` — Audubon bird names still in current use
* `datasets/elicitation/ft_modern_american_birds.jsonl` — Modern bird names (LLM-generated)

**Mitigation/ablation variants** — generated by `python generate_datasets.py --experiment birds`:
* `datasets/mitigations/relevant/` — context prefixes plausibly relevant to 19th-century birds
* `datasets/mitigations/irrelevant/` — context prefixes irrelevant to the training domain
* `datasets/ablations/user_system_prompt/` — user vs. system prompt placement ablation
* `datasets/ablations/time-incremental/` — time prefix varying from 1850 to 2050

## Training

See `../sft/configs/elicitation/birds/` for all training configs.
Training configs for mitigation experiments are in `../sft/configs/mitigation/birds/`.

## Evaluation

Evaluation questions and judge prompts are in [evaluation/questions.yaml](evaluation/questions.yaml).

The experiment-level evaluate.py uses `llmcomp` to evaluate multiple models at once:
```bash
pip install llmcomp==1.2.1
# Edit the MODELS variable in evaluation/evaluate.py, then:
python evaluation/evaluate.py
```

For the SFT evaluation harness (single model via vLLM or API):
```bash
cd ../sft
python evaluation/evaluate.py \
    --experiment birds \
    --model-base-url http://localhost:PORT/v1 \
    --output-dir results/birds/llama-3.1-8B-r4-1ep
```
